StorageSession.CommitObjectState Method

Some times it is more useful to use the CommitObjectState method to save the object state, than use the object state transition technique from transactions system. The call of CommitObjectState method is more risky, because if something goes wrong when you change the object state the object can't roll back to original state and program can't reach to the code that call the CommitObjectState method.
